01.git

Posted at 2018-11-29 git 

1.git pull 或者 git clone 从远程分支拉取项目。

  • 如果远程已经有对应的分支

    git checkout -b 新分支 origin/远程分支

  • 如果远程没有对应的分支

​ git checkout -b 新分支,在新分支里开发代码

3.git checkout master，开发完成后，回到主分支

4.git pull ，拉取最新的项目代码（方便在本地解决冲突）

  • 此时如果远程已经发生变化会pull失败
  • git stash [save tag名称]
  • git pull
  • 如果提示冲突，则需要解决冲突

git add .

git commit -m “”

git push –set-upstream origin 远程分支名称

//–set-upstream的作用是和远程分支关联，下次无需再指定分支名称

5.git checkout 新分支,回到新分支

6.git merge master,将主分支代码合并过来，解决冲突

将远程仓库与本地进行关联(在上传代码之前)

git remote add origin 仓库地址

7.git push(-u) origin 远程新分支,将代码上传到远程分支.

git fetch（地址） 本地分支名:远程分支名

解决本地仓库没有readme无法提交问题

git pull –rebase origin master

//先回退本地分支

git reflog

git reset –hard Obfafd (Head表示当前版本，Head^上一个版本)

//再回退远程分支

git push -f origin maste 这里假设只有一个master分支

  • 第一次上传
  • 第一次上传后的修改及以后上传的操作
